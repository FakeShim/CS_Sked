Frequently Asked Questions - Brandon Nguyen

If I had to change out the user email of this application, how would I do that?

As there is currently no real way of creating new accounts, you would need permissions to go into the database wherever the login information is stored and clear out the login table. Then when you sign into the application, the next email and password should populate it. Note: As stated previously, only one user should populate this table

What should I do if I receive an error when logging in or creating an account?

You would need to make sure you enter the right user email and password. If you do not remember the user email and password, you would need to get permission to repopulate the user email and password in the database that it is stored in. I f that happens, look back at the previous question to see how that is done.

What should I do if the recepients don't see any emails from the application?

You should check with the recipients to see if the email that shows on the table their correct email.

If it is, then tell the recipients that it may show up in their spam folder due to the formatting of the email.

If not, you can change the contact of the faculty member in the faculty page. Then resend the email by re-entering the request information and pressing the submit button under the available faculty table.

What should I do to check to see that MongoDB is working on Docker after migration or installation?

After installation, you can check to see if you have a MongoDB Atlas collection by running: DB_USERNAME=db-username DB_PASSWORD=db-password docker-compose build.

After that you should run:

DB_USERNAME=db-username DB_PASSWORD=db-password docker-compose up

If this does not work, then you would need to set up a MongoDB Atlas database here

Important reminders when creating a database for this application:

  • Follow the schema that we created
    • There are three collections: faculty, login, and requests
      • In faculty, there are 5 fields: id, email, availability (which is an object), faculty's first name, and faculty's last name
      • The availability object should have 5 arrays named after the days of the week. In those arrays, you would need to fill it in with 12 boolean values. They represent the empty time slots that a faculty member will be available for.
      • In login, there are 3 fields: id, email, and the hashed password. However, there is no real way to really create a new account as addressed in the modify page
      • In requests, there should be 8 fields: id, faculty's first name, faculty's last name, student's first name, student's last name, email, status, and times (an object)
        • Times should be an array of strings which holds the times
    • Remember the user email and password used to login into Atlas to see the database changes
If they are not using Atlas, they can follow the instructions of how to use a local MongoDB database detailed in the extensions page.
