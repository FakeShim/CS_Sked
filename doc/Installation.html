How to Install Software - Peter Shim


In order for someone to deploy this application on their own environment, they would first need to clone our GitHub repository.

They could use this command in their terminal:

git clone git@github.com:FakeShim/CS_Sked.git

Or they could download the GitHub desktop app and clone the repository through the desktop app.


Next they would need to install Docker

To install Docker, follow this website:

https://docs.docker.com/get-docker/

Make sure to follow the instructions in the provided documentation that corresponds to your device and reopen the app after setting it up.

Once Docker is up and running use the following commands in a bash terminal or WSL:


A MongoDB atlas collection will be necessary after this.

If you already have one, then make sure to keep track of the DB username and password

If not, check the FAQ's on how to set up a MongoDB atlas collection

DB_USERNAME=db-username DB_PASSWORD=db-password docker-compose build

The user will need to make sure to replace db-username and db-password with the correct fields for their MongoDB Database.


How to run locally (using Docker)

If you plan to run the project locally you will have to change a few environment variables within the files.

Everything needed for building the docker images with docker-compose will be in the docker-compose.yaml file. It shows the ports, environment variables, and directories for the project.

You will also need to change the file Dockerfile.web in:

server/

And then rename it to just Dockerfile

In order for this to be ran locally, you will also need to change the const variable backend_host value to 'localhost:443'

The following files will need these changes:

client/src/components/App.js

client/src/components/Confirmation.js

client/src/components/Faculty.js

client/src/components/Home.js

client/src/components/Login.js

client/src/components/Requests.js

client/src/components/table.js

Once backend_host has been changes in the above files, you can now run the following Docker commands

DB_USERNAME=db-username DB_PASSWORD=db-password docker-compose build

and

DB_USERNAME=db-username DB_PASSWORD=db-password docker-compose up

Replace db-username and db-password just like in the previous section.

This will take a while, as it is installing many packages.

If it reaches a runtime error, you will just have to run this command again:

DB_USERNAME=db-username DB_PASSWORD=db-password docker-compose up

Once the project is running, you can access it by clicking on the link in the terminal:

Local: https://localhost:443

To end the program just use the hotket ctrl-c


Heroku install for running globally(Front-end used the OIT server provided).

Here is how to set up Heroku in the terminal:

Login:

heroku container:login

Push Docker image to Heroku:

heroku container:push --recursive -a your-app-name

your-app-name being the name of your Heroku app

Release your Docker Image:

heroku container:release web -a your-app-name

Viewing your app:

heroku open -a your-app-name
